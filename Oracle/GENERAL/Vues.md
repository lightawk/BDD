## Vues V$

-----------

> Vues concernant la journalisation.

```
| Vue            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|:--------------:|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------:|
| v$log          | Affiche les informations des fichiers journaux provenant du fichier de controle                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| v$log_history  | Affiche les informations de l'historique des fichiers de journalisation provenant du fichier de controle                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| v$logfile      | Contient des informations a propos des fichiers journaux                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| v$archived_log | Affiche les informations de journal archivees a partir du fichier de controle y compris les noms de journal d'archivage. Un enregistrement de journal d'archivage est insere après que le journal de retablissement en ligne est correctement archive ou efface, la colonne de nom est "null" si le journal a ete efface. Si le journal est archivé deux fois, il y aura deux enregistrements de journal archivés avec les mêmes THREAD #, SEQUENCE # et FIRST_CHANGE #, mais avec un nom différent. Un enregistrement de journal d'archivage est également inséré lorsqu'un journal d'archivage est restauré à partir d'un jeu de sauvegarde ou d'une copie et chaque fois qu'une copie d'un journal est réalisée à l'aide de la commande RMAN COPY) |
```

```
| Vue             | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|:---------------:|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------:|

| v$sqlarea       | Repertorie les statistiques sur la zone SQL partagée et contient une ligne par chaîne SQL. Il fournit des statistiques sur les instructions SQL en mémoire, analysées et prêtes à être exécutées                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| v$sql           | Repertorie les statistiques sur la zone SQL partagée sans la clause GROUP BY et contient une ligne pour chaque enfant du texte SQL d'origine entré. Les statistiques affichées dans V$SQL sont normalement mises à jour à la fin de l'exécution de la requête. Toutefois, pour les requêtes de longue durée, elles sont mises à jour toutes les 5 secondes. Cela permet de voir facilement l'impact des instructions SQL de longue durée lorsqu'elles sont toujours en cours)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| v$transaction   | Liste des transactions actives dans le système)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| v$bh            | Affiche l'état et le nombre de pings pour chaque tampon dans le SGA. Ceci est une vue Real Application Clusters)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| v$dbfile        | Affiche tous les fichiers de données composant la base de données. Cette vue est conservée pour la compatibilité historique. L'utilisation de V$DATAFILE est recommandée à la place)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| v$undostat      | Affiche un histogramme de données statistiques indiquant le fonctionnement du système. Les statistiques disponibles incluent la consommation d'espace d'annulation, la simultanéité des transactions et la longueur des requêtes exécutées dans l'instance. Vous pouvez utiliser cette vue pour estimer la quantité d'espace d'annulation nécessaire pour le workload actuel. Oracle utilise cette vue pour régler l'utilisation de l'annulation dans le système. La vue renvoie des valeurs NULL si le système est en mode de gestion d'annulation manuelle. Chaque ligne de la vue conserve les statistiques collectées dans l'instance pendant un intervalle de 10 minutes. Les lignes sont dans l'ordre décroissant en fonction de la valeur de la colonne BEGIN_TIME. Chaque ligne appartient à l'intervalle de temps marqué par (BEGIN_TIME, END_TIME). Chaque colonne représente les données collectées pour la statistique particulière dans cet intervalle de temps. La première ligne de la vue contient des statistiques pour la période actuelle (partielle). La vue contient un total de 1008 lignes, couvrant un cycle de 7 jours) |
| v$rollstat      | Contient les statistiques des rollback segments                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| v$sort_segment  | Contient des informations sur chaque segment de tri dans une instance donnee. La vue n'est mise à jour que lorsque le tablespace est du type TEMPORARY)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

| v$tempseg_usage | Decrit l'utilisation du segment temporaire                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
```
